Los endpoints de Ventas te permiten obtener todo tu historial de transacciones de SmartPyme —incluidas las líneas de detalle individuales, los datos del cliente y la información de pago— y crear o actualizar ventas desde sistemas externos como tiendas e-commerce, marketplaces o integraciones de POS. Úsalos para construir dashboards de reportes, sincronizar transacciones de forma bidireccional, auditar registros completados y pendientes o generar cotizaciones de forma programática. Todas las solicitudes deben incluir tu API key en el encabezado `Authorization`.

## GET /sales

Devuelve una lista paginada de registros de ventas. Aplica parámetros de consulta para acotar los resultados por rango de fechas, estado o criterio de orden. Cada registro en la respuesta incluye el objeto Venta completo junto con sus líneas `detalles` asociadas.

<CodeGroup>

```bash cURL
curl --request GET \
  --url "https://api.smartpyme.site/api/external/v1/sales?fecha_inicio=2025-01-01&fecha_fin=2025-01-31&estado=Completada&per_page=50" \
  --header "Authorization: Bearer {api_key}"
```

```python Python
import requests

url = "https://api.smartpyme.site/api/external/v1/sales"
headers = {"Authorization": "Bearer {api_key}"}
params = {
    "fecha_inicio": "2025-01-01",
    "fecha_fin": "2025-01-31",
    "estado": "Completada",
    "per_page": 50,
}

response = requests.get(url, headers=headers, params=params)
print(response.json())
```

```javascript JavaScript
const response = await fetch(
  "https://api.smartpyme.site/api/external/v1/sales" +
    "?fecha_inicio=2025-01-01&fecha_fin=2025-01-31&estado=Completada&per_page=50",
  {
    method: "GET",
    headers: { Authorization: "Bearer {api_key}" },
  }
);
const data = await response.json();
console.log(data);
```

</CodeGroup>

## POST /sales

Crea y procesa una venta usando la misma lógica de facturación que utiliza la interfaz de SmartPyme. Usa este endpoint para registrar transacciones en SmartPyme desde sistemas externos de POS, e-commerce o marketplaces, o para generar cotizaciones de forma programática. El endpoint ejecuta toda la canalización de negocio —movimientos de stock, desglose de impuestos, correlativos de documento y generación del documento de venta— así que la venta creada por este endpoint es indistinguible de una capturada manualmente en la aplicación.

### Cuándo usarlo

- Necesitas registrar una venta confirmada y marcarla como pagada (`Pagada` o `Completada`) de inmediato.
- Necesitas abrir una venta pendiente para finalizarla más tarde desde la interfaz de SmartPyme (`Pendiente`).
- Necesitas generar una cotización desde una herramienta externa (`Cotizacion`).

### Idempotencia

Envía un identificador externo en `referencia_externa` (o `referencia`) para hacer la solicitud idempotente. Si ya existe una venta para tu empresa con la misma `referencia`, la API devuelve el registro existente con HTTP `200` y `meta.idempotent: true` en lugar de crear un duplicado. Las ventas nuevas devuelven HTTP `201`. Configura siempre este campo cuando reintentes solicitudes o cuando sincronices desde sistemas que puedan reenviar la misma transacción.

<CodeGroup>

```bash cURL
curl --request POST \
  --url "https://api.smartpyme.site/api/external/v1/sales" \
  --header "Authorization: Bearer {api_key}" \
  --header "Content-Type: application/json" \
  --data '{
    "referencia_externa": "ORDER-9001",
    "fecha": "2025-01-15",
    "estado": "Pagada",
    "id_sucursal": 3,
    "id_bodega": 7,

    "id_canal": 2,
    "id_cliente": 45,
    "forma_pago": "Efectivo",
    "monto_pago": 180.00,

    "observaciones": "Pedido web",

      {

        "cantidad": 1,
        "precio": 163.75,
        "descuento": 10.00,
        "porcentaje_impuesto": 13
      }
    ]
  }'
```

### Respuesta

En caso de éxito, la respuesta envuelve el objeto Venta completo bajo `data`, con la misma estructura que devuelve [GET /sales](#get-sales). La bandera `meta.idempotent` indica si el registro se creó en esta solicitud o se devolvió de una solicitud anterior:

- `201 Created` — se creó una venta nueva (`meta.idempotent: false`).
- `200 OK` — la solicitud coincidió con una `referencia` existente y se devuelve la venta creada previamente (`meta.idempotent: true`).

```json
{
  "success": true,
  "data": {
    "id": 10422,
    "fecha": "2025-01-15",
    "correlativo": "FAC-002502",
    "estado": "Completada",
    "total": 180.00,
    "detalles": [ /* ... */ ]
  },
  "meta": {
    "empresa": "Mi Empresa S.A.",
    "timestamp": "2025-01-15T10:42:00Z",
    "idempotent": false
  }
}
```

Los errores de validación o de reglas de negocio devuelven la envoltura de error estándar con HTTP `422` y un objeto `details` con los campos en falta.

Recupera un único registro de venta por su ID numérico. La respuesta devuelve el mismo objeto Venta completo descrito anteriormente, incluido el arreglo `detalles` con las líneas de detalle.

## PUT /sales/\{id\}

Actualiza una venta o cotización existente que aún no se ha finalizado. Usa este endpoint para ajustar líneas de detalle, cambiar el cliente, cambiar la sucursal o bodega, o convertir una cotización en una venta confirmada.

### Cuándo usarlo

- Editar un registro en estado `Pendiente`, `Cotizacion` o `Pre-venta` antes de finalizarlo —por ejemplo, para agregar o quitar productos, corregir el cliente o cambiar la forma de pago.
- Convertir una cotización en venta enviando `estado: "Pagada"` o `"Completada"` junto con las líneas finales.

### Restricciones

- Solo se pueden actualizar ventas en estado `Pendiente`, `Cotizacion` o `Pre-venta`. Intentar actualizar una venta en otro estado devuelve HTTP `422`.
- No se pueden actualizar ventas que ya tienen un documento fiscal emitido; devuelven HTTP `422`.
- No se pueden actualizar ventas anuladas (`Anulada`); devuelven HTTP `422`.
- Se aplica el mismo máximo de 100 líneas al arreglo `detalles`.

### Parámetro de ruta

<ParamField path="id" type="integer" required>
  Identificador numérico de la venta a actualizar.
</ParamField>

### Parámetros del cuerpo

Todos los campos del cuerpo son opcionales. Envía solamente los campos que quieras modificar; los campos omitidos conservan su valor actual. Si se omite `detalles`, se conservan las líneas existentes.

<ParamField body="fecha" type="string">

## GET /sales/summary

Devuelve estadísticas agregadas para tus datos de ventas sobre un rango de fechas opcional. Usa este endpoint para construir widgets de reportes de alto nivel: ingreso total, valor promedio de transacción, impuestos recaudados y un desglose de conteos y totales de transacciones por estado.

<CodeGroup>

```bash cURL
curl --request GET \
  --url "https://api.smartpyme.site/api/external/v1/sales/summary?fecha_inicio=2025-01-01&fecha_fin=2025-01-31" \
  --header "Authorization: Bearer {api_key}"
```

```python Python
import requests

url = "https://api.smartpyme.site/api/external/v1/sales/summary"
headers = {"Authorization": "Bearer {api_key}"}
params = {
    "fecha_inicio": "2025-01-01",
    "fecha_fin": "2025-01-31",
}

response = requests.get(url, headers=headers, params=params)
print(response.json())
```

```javascript JavaScript
const response = await fetch(
  "https://api.smartpyme.site/api/external/v1/sales/summary" +

    method: "GET",
    headers: { Authorization: "Bearer {api_key}" },
  }

const data = await response.json();
console.log(data);
```
