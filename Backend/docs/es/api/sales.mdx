Los endpoints de Ventas te permiten obtener todo tu historial de transacciones de SmartPyme, incluidas las líneas de detalle individuales, los datos del cliente y la información de pago. Usa estos endpoints para crear dashboards de reportes, sincronizar datos con sistemas externos o auditar transacciones completadas y pendientes. Todas las solicitudes deben incluir tu API key en el encabezado `Authorization`.

---

## GET /sales

Devuelve una lista paginada de registros de ventas. Aplica parámetros de consulta para acotar los resultados por rango de fechas, estado o criterio de orden. Cada registro en la respuesta incluye el objeto Venta completo junto con sus líneas `detalles` asociadas.

<CodeGroup>

```bash cURL
curl --request GET \
  --url "https://api.smartpyme.site/api/external/v1/sales?fecha_inicio=2025-01-01&fecha_fin=2025-01-31&estado=Completada&per_page=50" \
  --header "Authorization: Bearer {api_key}"
```

```python Python
import requests

url = "https://api.smartpyme.site/api/external/v1/sales"
headers = {"Authorization": "Bearer {api_key}"}
params = {
    "fecha_inicio": "2025-01-01",
    "fecha_fin": "2025-01-31",
    "estado": "Completada",
    "per_page": 50,
}

response = requests.get(url, headers=headers, params=params)
print(response.json())
```

```javascript JavaScript
const response = await fetch(
  "https://api.smartpyme.site/api/external/v1/sales" +
    "?fecha_inicio=2025-01-01&fecha_fin=2025-01-31&estado=Completada&per_page=50",
  {
    method: "GET",
    headers: { Authorization: "Bearer {api_key}" },
  }
);
const data = await response.json();
console.log(data);
```

</CodeGroup>

### Parámetros de consulta

<ParamField query="fecha_inicio" type="string">
  Fecha de inicio para filtrar ventas, en formato `Y-m-d` (por ejemplo, `2025-01-01`). Se usa junto con `fecha_fin` para definir un rango de fechas.
</ParamField>

<ParamField query="fecha_fin" type="string">
  Fecha de fin para filtrar ventas, en formato `Y-m-d` (por ejemplo, `2025-01-31`). Se usa junto con `fecha_inicio` para definir un rango de fechas.
</ParamField>

<ParamField query="estado" type="string">
  Filtra por estado de la venta. Valores aceptados: `Completada`, `Pendiente`, `Anulada`, `Cotizacion`.
</ParamField>

<ParamField query="page" type="integer">
  Número de página a obtener. Su valor por defecto es `1`.
</ParamField>

<ParamField query="per_page" type="integer">
  Cantidad de registros por página. Acepta valores entre `1` y `200`. Su valor por defecto es `100`.
</ParamField>

<ParamField query="order_by" type="string">
  Campo por el cual ordenar los resultados. Valores aceptados: `fecha`, `total`, `correlativo`, `created_at`.
</ParamField>

<ParamField query="order_direction" type="string">
  Dirección del orden. Valores aceptados: `asc`, `desc`.
</ParamField>

## GET /sales/\{id\}

Recupera un único registro de venta por su ID numérico. La respuesta devuelve el mismo objeto Venta completo descrito anteriormente, incluido el arreglo `detalles` con las líneas de detalle.

### Parámetro de ruta

<ParamField path="id" type="integer" required>
  El identificador numérico único de la venta que deseas obtener.
</ParamField>

<CodeGroup>

```bash cURL
curl --request GET \
  --url "https://api.smartpyme.site/api/external/v1/sales/10421" \
  --header "Authorization: Bearer {api_key}"
```

```python Python
import requests

sale_id = 10421
url = f"https://api.smartpyme.site/api/external/v1/sales/{sale_id}"
headers = {"Authorization": "Bearer {api_key}"}

response = requests.get(url, headers=headers)
print(response.json())
```

```javascript JavaScript
const saleId = 10421;
const response = await fetch(
  `https://api.smartpyme.site/api/external/v1/sales/${saleId}`,
  {
    method: "GET",
    headers: { Authorization: "Bearer {api_key}" },
  }
);
const data = await response.json();
console.log(data);
```

</CodeGroup>

La respuesta envuelve el objeto Venta bajo una clave `data` con `"success": true`. Todos los campos, incluido `detalles`, son idénticos a la estructura documentada en la respuesta de [GET /sales](#get-sales) anterior.

## GET /sales/summary

Devuelve estadísticas agregadas para tus datos de ventas sobre un rango de fechas opcional. Usa este endpoint para construir widgets de reportes de alto nivel: ingreso total, valor promedio de transacción, impuestos recaudados y un desglose de conteos y totales de transacciones por estado.

<CodeGroup>

```bash cURL
curl --request GET \
  --url "https://api.smartpyme.site/api/external/v1/sales/summary?fecha_inicio=2025-01-01&fecha_fin=2025-01-31" \
  --header "Authorization: Bearer {api_key}"
```

```python Python
import requests

url = "https://api.smartpyme.site/api/external/v1/sales/summary"
headers = {"Authorization": "Bearer {api_key}"}
params = {
    "fecha_inicio": "2025-01-01",
    "fecha_fin": "2025-01-31",
}

response = requests.get(url, headers=headers, params=params)
print(response.json())
```

```javascript JavaScript
const response = await fetch(
  "https://api.smartpyme.site/api/external/v1/sales/summary" +
    "?fecha_inicio=2025-01-01&fecha_fin=2025-01-31",
  {
    method: "GET",
    headers: { Authorization: "Bearer {api_key}" },
  }
);
const data = await response.json();
console.log(data);
```

</CodeGroup>

### Parámetros de consulta

<ParamField query="fecha_inicio" type="string">
  Fecha de inicio del período de resumen, en formato `Y-m-d`. Omítela para incluir todos los registros desde el inicio.
</ParamField>

<ParamField query="fecha_fin" type="string">
  Fecha de fin del período de resumen, en formato `Y-m-d`. Omítela para incluir todos los registros hasta la fecha actual.
</ParamField>

<ParamField query="estado" type="string">
  Limita el resumen a un estado de venta específico. Valores aceptados: `Completada`, `Pendiente`, `Anulada`, `Cotizacion`.
</ParamField>

### Campos de la respuesta

<ResponseField name="data.cantidad_ventas" type="integer">
  Número total de registros de venta dentro del período solicitado.
</ResponseField>

<ResponseField name="data.total_ventas" type="number">
  Ingresos brutos en todas las ventas del período.
</ResponseField>

<ResponseField name="data.total_iva" type="number">
  Impuesto total recaudado en todas las ventas del período.
</ResponseField>

<ResponseField name="data.total_descuentos" type="number">
  Monto total de descuentos aplicados en todas las ventas del período.
</ResponseField>

<ResponseField name="data.promedio_venta" type="number">
  Valor promedio de venta para el período, calculado como `total_ventas / cantidad_ventas`.
</ResponseField>

<ResponseField name="data.ventas_por_estado" type="array">
  Desglose del conteo y el ingreso total de ventas agrupado por estado.

  <Expandable title="Campos de ventas_por_estado">
    <ResponseField name="estado" type="string">
      El grupo de estado de la venta (por ejemplo, `Completada`, `Pendiente`, `Anulada`, `Cotizacion`).
    </ResponseField>
    <ResponseField name="cantidad" type="integer">
      Número de ventas en este grupo de estado.
    </ResponseField>
    <ResponseField name="total" type="number">
      Valor total combinado de las ventas en este grupo de estado.
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="meta.empresa" type="string">
  Nombre visible de la empresa asociada a tu API key.
</ResponseField>

<ResponseField name="meta.timestamp" type="string">
  Marca de tiempo en formato ISO 8601 que indica cuándo se generó el resumen.
</ResponseField>

<ResponseField name="meta.filters_applied" type="object">
  Eco de los filtros utilizados para calcular el resumen, útil para confirmar los parámetros de la solicitud.
</ResponseField>
