Los endpoints de Inventario te dan acceso programático a tu catálogo completo de productos y a los datos de stock por bodega en SmartPyme. Puedes filtrar por categoría, marca o estado de stock para crear integraciones como alertas de stock bajo, sincronizaciones de catálogo para e-commerce o reportes de costo de mercadería. Todas las solicitudes deben incluir tu API key en el encabezado `Authorization`.

---

## GET /inventory

Devuelve una lista paginada de productos de tu catálogo. Cada registro de producto incluye precio, costo y un arreglo `inventarios` que muestra los niveles de stock actuales desglosados por bodega y sucursal.

<CodeGroup>

```bash cURL
curl --request GET \
  --url "https://api.smartpyme.site/api/external/v1/inventory?marca=Dell&con_stock=true&per_page=50" \
  --header "Authorization: Bearer {api_key}"
```

```python Python
import requests

url = "https://api.smartpyme.site/api/external/v1/inventory"
headers = {"Authorization": "Bearer {api_key}"}
params = {
    "marca": "Dell",
    "con_stock": True,
    "per_page": 50,
}

response = requests.get(url, headers=headers, params=params)
print(response.json())
```

```javascript JavaScript
const response = await fetch(
  "https://api.smartpyme.site/api/external/v1/inventory" +
    "?marca=Dell&con_stock=true&per_page=50",
  {
    method: "GET",
    headers: { Authorization: "Bearer {api_key}" },
  }
);
const data = await response.json();
console.log(data);
```

</CodeGroup>

### Parámetros de consulta

<ParamField query="codigo" type="string">
  Filtra por SKU o código interno del producto. Realiza una coincidencia parcial.
</ParamField>

<ParamField query="nombre" type="string">
  Filtra por nombre del producto. Realiza una coincidencia parcial sin distinguir mayúsculas y minúsculas.
</ParamField>

<ParamField query="categoria" type="string">
  Filtra por nombre de categoría. Realiza una coincidencia parcial contra `nombre_categoria`.
</ParamField>

<ParamField query="marca" type="string">
  Filtra por nombre de marca. Realiza una coincidencia parcial.
</ParamField>

<ParamField query="tipo" type="string">
  Filtra por tipo de producto. Valores aceptados: `Producto`, `Servicio`.
</ParamField>

<ParamField query="enable" type="integer">
  Filtra por estado activo. Usa `1` para productos activos y `0` para productos inactivos.
</ParamField>

<ParamField query="con_stock" type="boolean">
  Cuando es `true`, devuelve solo los productos con stock mayor a cero en al menos una bodega.
</ParamField>

<ParamField query="stock_minimo" type="boolean">
  Cuando es `true`, devuelve solo los productos cuyo stock actual está en o por debajo del umbral de stock mínimo configurado.
</ParamField>

<ParamField query="page" type="integer">
  Número de página a obtener. Su valor por defecto es `1`.
</ParamField>

<ParamField query="per_page" type="integer">
  Cantidad de registros por página. Acepta valores entre `1` y `200`. Su valor por defecto es `100`.
</ParamField>

<ParamField query="order_by" type="string">
  Campo por el cual ordenar los resultados. Valores aceptados: `nombre`, `codigo`, `precio`, `costo`, `created_at`.
</ParamField>

<ParamField query="order_direction" type="string">
  Dirección del orden. Valores aceptados: `asc`, `desc`.
</ParamField>

## GET /inventory/\{id\}

Recupera un único registro de producto por su ID numérico. La respuesta devuelve el mismo objeto Producto completo descrito anteriormente, incluido el arreglo `inventarios` con los niveles de stock por bodega.

### Parámetro de ruta

<ParamField path="id" type="integer" required>
  El identificador numérico único del producto que deseas obtener.
</ParamField>

<CodeGroup>

```bash cURL
curl --request GET \
  --url "https://api.smartpyme.site/api/external/v1/inventory/301" \
  --header "Authorization: Bearer {api_key}"
```

```python Python
import requests

product_id = 301
url = f"https://api.smartpyme.site/api/external/v1/inventory/{product_id}"
headers = {"Authorization": "Bearer {api_key}"}

response = requests.get(url, headers=headers)
print(response.json())
```

```javascript JavaScript
const productId = 301;
const response = await fetch(
  `https://api.smartpyme.site/api/external/v1/inventory/${productId}`,
  {
    method: "GET",
    headers: { Authorization: "Bearer {api_key}" },
  }
);
const data = await response.json();
console.log(data);
```

</CodeGroup>

La respuesta envuelve el objeto Producto bajo una clave `data` con `"success": true`. Todos los campos, incluido `inventarios`, son idénticos a la estructura documentada en la respuesta de [GET /inventory](#get-inventory) anterior.

## GET /inventory/summary

Devuelve estadísticas agregadas para todo tu catálogo de productos y la posición combinada de stock. Usa este endpoint para obtener una vista rápida del estado del catálogo: cuántos productos están activos, el stock total disponible, el valor total del inventario y qué categorías contienen más productos.

<CodeGroup>

```bash cURL
curl --request GET \
  --url "https://api.smartpyme.site/api/external/v1/inventory/summary" \
  --header "Authorization: Bearer {api_key}"
```

```python Python
import requests

url = "https://api.smartpyme.site/api/external/v1/inventory/summary"
headers = {"Authorization": "Bearer {api_key}"}

response = requests.get(url, headers=headers)
print(response.json())
```

```javascript JavaScript
const response = await fetch(
  "https://api.smartpyme.site/api/external/v1/inventory/summary",
  {
    method: "GET",
    headers: { Authorization: "Bearer {api_key}" },
  }
);
const data = await response.json();
console.log(data);
```

</CodeGroup>

### Campos de la respuesta

<ResponseField name="data.productos.total" type="integer">
  Número total de productos en tu catálogo, sin importar su estado.
</ResponseField>

<ResponseField name="data.productos.activos" type="integer">
  Número de productos actualmente marcados como activos (`enable = 1`).
</ResponseField>

<ResponseField name="data.productos.inactivos" type="integer">
  Número de productos actualmente marcados como inactivos (`enable = 0`).
</ResponseField>

<ResponseField name="data.inventario.stock_total" type="number">
  Suma de todas las cantidades de stock en todos los productos y bodegas.
</ResponseField>

<ResponseField name="data.inventario.valor_total" type="number">
  Valor total del inventario calculado usando el costo promedio de cada producto (`costo_promedio`) multiplicado por su stock actual.
</ResponseField>

<ResponseField name="data.inventario.productos_con_stock" type="integer">
  Número de productos distintos que tienen al menos una unidad en stock en alguna bodega.
</ResponseField>

<ResponseField name="data.inventario.productos_sin_stock" type="integer">
  Número de productos distintos con stock cero en todas las bodegas.
</ResponseField>

<ResponseField name="data.inventario.productos_stock_bajo" type="integer">
  Número de productos cuyo stock actual está en o por debajo del umbral `stock_minimo` configurado en al menos una bodega.
</ResponseField>

<ResponseField name="data.productos_por_categoria" type="array">
  Lista de conteos de productos agrupados por categoría.

  <Expandable title="Campos de productos_por_categoria">
    <ResponseField name="categoria" type="string">
      Nombre de la categoría de productos.
    </ResponseField>
    <ResponseField name="cantidad" type="integer">
      Número de productos asignados a esta categoría.
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="meta.empresa" type="string">
  Nombre visible de la empresa asociada a tu API key.
</ResponseField>

<ResponseField name="meta.timestamp" type="string">
  Marca de tiempo en formato ISO 8601 que indica cuándo se generó el resumen.
</ResponseField>

<ResponseField name="meta.filters_applied" type="object">
  Eco de los filtros utilizados para calcular el resumen.
</ResponseField>
